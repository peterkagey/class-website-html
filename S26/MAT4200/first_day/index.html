First day activity

Drawing with SVG (Scalable Vector Graphics) We're going to start with the following SVG code (you can scroll right to see the whole thing): 
<svg xmlns="http://www.w3.org/2000/svg" width="300" height="300" viewbox="0 0 100 100" fill="none">
  <rect x="0" y="0" height="100" width="100" fill="black"/>
  <path d="M 20 90 L 80 40 L 20 50 L 20 20" stroke="white" stroke-width="2px"/>
</svg>
Here's how we understand it:
  • The first line (beginning <svg xmlns...) is just telling the SVG parser that we're talking about an SVG file.
  • The width="300" height="300" variables tell your browser to display it as 300px × 300px.
  • The viewbox="0 0 100 100" tells us that we're dealing with coordinates between 0 and 100 for both the $x$ and $y$ axis. (Mathematically, this means we're drawing in the square $[0,100] \times [0,100] \subsetneq \mathbb{R}^2$)
  • The code fill="none" sets a default telling the parser not to fill in the shapes and only draw the outlines.
  • The line beginning <rect ... draws a black background starting at $(0,0)$ in the upper and ending at $(100,100)$ in the lower right. (Note that the $y$ coordinate is flipped!)
  • The next line (which begins <path d=) is where the magic happens. The d stands for data, and describes how to draw the path.
    • Inside of the string, M stands for Move, so the code M 20 90 says to move to the position $(20,90)$. (Remember, the $y$-coordinate points down!)
    • The L stands for Line, so the code L 80 40 says to draw a line from the previous position $(20, 90)$ to the specified position $(80, 40)$.
  • On the path line, stroke="white" stroke-width="2px" specifies the color and thickness of the line.
  • The final </svg> is a "closing tag" which just tells the parser that we're all done!

Let's draw!

You'll hear me say this term that the best way to learn mathematics (and lots of other things) is to just play!

Start by changing some numbers and colors, and see what happens. As you change the code, the SVG image should automatically update.

The coordinate system

Recall that the coordinate system that SVG files use is "updside down" because it draws the pixels in reading order (left-to-right and top-to-bottom.) This means that the $x$-axis points to the right ➡️, and the $y$-axis points down ⬇️, which can take some getting used to! (One unexpected consequence of this is that we measure angles clockwise instead of counterclockwise!)

Do the following activities with your group, and reach out if you get stuck!

  1. Drawing with lines
    1. On a piece of paper or tablet or whiteboard, create a logo which consists of three to six lines.
    2. Try reproduce it as an SVG as accurately as possible. (If you have multiple colors, you can draw multiple path elements, you can also draw lines on top of each other with different thicknesses.)
    3. Take a picture of your drawing and a screenshot of the SVG and post them both to the #in-class Discord channel. (You can find the invitation to the Discord channel on our syllabus.)
  2. Parameterizing lines
    1. The standard way to parameterize a line from a point (vector) $\mathbf{v}_1$ to a point $\mathbf{v}_2$ is with the formula \[ \boldsymbol{\ell}(t) = (1-t)\mathbf{v}_1 + t\mathbf{v}_2, \] for $t \in [0,1]$. Check the endpoints by evaluating at $t = 0$ and $t=1$ to see that this gives what you want.
    2. Let $\mathbf{v}_1 = \begin{pmatrix}-1\\4\end{pmatrix}$ and $\mathbf{v}_2 = \begin{pmatrix}3\\2\end{pmatrix}$, and write the parameterized curve in terms of its $x$- and $y$-coordinates: \[ \boldsymbol{\ell}(t) = \begin{pmatrix}\square\\\square\end{pmatrix}. \] (That is, fill in the squares.)
    3. In WolframAlpha, you can plot this as parametric plot (2t-1, 3t+2), t=0..1. Use WolframAlpha or any other tool to plot this line (but first try to imagine how you think it will look!)
  3. Drawing regular polygons.
    1. Draw an equilateral triangle in any orientation that you want.
    2. Optional challenge: draw the biggest possible equilateral triangle that will fit in the square.
    3. Draw a square, but draw it crooked so that the edges of the square aren't horizontal/vertical.
    4. Can you draw a regular pentagon?
  4. Drawing (Bézier) curves
    1. The most important element after for drawing nice smooth figures are the Quadratic and Cubic the Bézier curves.
    2. We'll start with the (simpler) quadratic bezier curve. In order to draw this we specify Q x1 y1 x y, where x1 y1 is called a "control point" and x y is the endpoint. Append Q 50 20 70 80 to the end of your code, wiggle the numbers, and see if you can figure out what's going on.
    3. Here's something that might help you figure out what's going on: right now your path should end with x' y' Q x1 y1 x y. Write a new line beneath this which is 
      <path d="M x' y' L x1 y1 L x y" stroke="red" stroke-dasharray="1"/>
      Does this help you see what the control point is doing? (If this doesn't look right, let me know, and we can debug it together.)
    4. Now do the same thing with a Cubic Bézier curve, which has two control points, so we specify C x1 y1 x2 y2 x y. Choose numbers for each of these variables, append this to your SVG code, and see if you can sort out what's going on.
    5. We'll do something analogous to what we did before: your path should end with x' y' C x1 y1 x2 y2 x y. Write a new line beneath this which is 
      <path d="M x' y' L x1 y1 L x2 y2 L x y" stroke="green" stroke-dasharray="1"/>
      Does this help you see what the control points are doing?
    6. Try to connect two quadratic Bézier curves together so that the curve is "differentiable" (i.e. there's no "corner where they meet").
    7. Try to connect two cubic Bézier curves together so that the curve is "differentiable" (i.e. there's no "corner where they meet").
    8. Can you take your original line drawing "logo" and smooth out all of the corners by converting your lines to Bézier curves?
  5. (Bonus part) Drawing with circles and circular arcs.
    1. Drawing a circle is easy enough, just use code like <circle cx="50" cy="22" r="20" stroke="red">. (Put this into your SVG code)
    2. What do cx, cy, and r do? You can probably guess, but play around to find out!
    3. Maybe though, you want to use circular Arcs to make the corners of your logo look smooth. We don't use the circle tag for that, instead we use the A command in a path. This command is kind of complicated: 
      A rx ry x-axis-rotation large-arc-flag sweep-flag x y
      I'll explain what some of these things mean in a second, but start by putting A 75 75 0 0 0 50 50 at the end of one of your path elements and see what happens.
    4. This A instruction draws elliptical arcs, so it has you specify both the $x$- and $y$-radii. Since we're just thinking about circles, we have the rx and ry values the same, which means that the x-axis-rotation doesn't do anything (since rotating a circle looks the same). The large-arc-flag and sweep-flag> can be set to 0 or 1, and they just say which part of the circle to draw. Lastly the x and y at the end specify where to stop drawing the arc. Play around with these values and see if you can draw something interesting.

And beyond!

  1. Did you finish everything? If so, get more practice drawing with Bézier curves. See if you can write your name, draw a rainbow, approximate a circle with Bézier curves, etc.
  2. If you have experience programming (or even if you don't!) you can try to write a program that draws something more sophisticated with hundreds or thousands of lines, arcs, and Bézier curves.
  3. Look up how we parameterize a quadratic Bézier curve with starting point $\mathbf{v}_1$, control point $\mathbf{v}_2$, and endpoint $\mathbf{v}_3$. Can you prove that it has the properties that you think it does? How about a cubic Bézier curve?
