Exercise 2 [3]
  1. Install OpenSCAD.
  2. Run the following code. 
    color("red") {
    cylinder(20,4,4);
    translate([3.5,0,0]) cube([15,8,8],center=true);
    translate([-1,2,16]) linear_extrude(height = 3) text("r");
}
    Figure out how to make it display, rotate it around, zoom in, zoom out, etc. Describe it in words.
    3. Beneath that add this code, which will create some objects in blue defined by taking every single red point, and multiplying it by a matrix whose columns all have norm 1 and are orthogonal to each other. (Matrices with this property define the orthogonal group $O(3)$.) 
    color("blue")
multmatrix([
    [-0.707107, 0.408, 0.57735],
    [ 0.707107, 0.408, 0.57735],
    [ 0,        -0.816, 0.57735]
]) {
    cylinder(20,4,4);
    translate([3.5,0,0]) cube([15,8,8],center=true);
    translate([-1,2,16]) linear_extrude(height = 3) text("r");
}
    Describe in words what the matrix has done. In particular, what happened to the "r"?
  3. Now play around in OpenSCAD for at least ten minutes. Make something that you might want to 3D print. Export it as an STL file, and post the file together with a screenshot to the #hw-prompts Discord channel, and let us know if you used any tools/resources to generate the code.
Exercise 3 [7]
Recall that we have been modeling my Christmas tree in class, which we've been describing as a cone that is $7$ feet tall and $4$ feet in diameter at the base. In particular, we used the parameterization \[ \mathbf{c}(\theta, h) = \begin{pmatrix} \frac{2}{7}(7-h) \cos (\theta ) \\ \frac{2}{7}(7-h) \sin (\theta ) \\ h \end{pmatrix},\,\theta \in [0, 2\pi],\,h \in [0,7] \]
  1. Plot the tree in Mathematica with the following code, noting that I have renamed $\theta$ to $t$. 
    ParametricPlot3D[{2/7 (7 - h) Cos[t], 2/7 (7 - h) Sin[t], h}, {t, 0, 2 Pi}, {h, 0, 7}, PlotStyle -> Green]
    Save the image as a PNG, and include it in your write-up.
  2. Now we're going to add some ornaments to the tree. Save your plot of the tree as a variable by prepending treePlot = before ParametricPlot3D. Next we're going to make some ornaments named ornament1, ornament2, etc. to place on the tree. To make the ornament, use code like this 
    ornament1 = Graphics3D[{Red, Sphere[{1, 0, 3.5}, 0.2]}];
    which makes a red ornament at $(1,0,3.5)$ with a radius of $0.2$. Make a few more ornaments with different colors, positions, and radii, and place them on the tree with 
    Show[treePlot, ornament1, ..., ornament4]
    Save the image as a PNG, and include it in your write-up.
  3. By hand or with Mathematica, set up an integral to compute the surface area of the tree. Evaluate the integral by hand with a computer and report your answer as a decimal. (In Mathematica you can numerically integrate a function with NIntegrate.)
  4. Since I can't see ornaments at the back of my tree, I only place the ugly ones back there, and I have a higher density of ornaments at the front. We will model the density of ornaments by $4\frac{(\theta - \pi)^2}{\pi^2}$ ornaments per square foot, where $\theta$ is the angle from the front of the tree.
    1. "The zeroth lesson in mathematics is you should always begin by guessing!"1 Imagine a tree of the size described, and then imagine four ornaments per square foot at the front. What would you guess is the number of ornaments on the tree?
    2. Set up an integral to compute the number of ornaments on the tree according to this model. Could this possibly be the number of ornaments I actually had on my tree?
    3. How many ornaments do I have in the $90^\circ$ wedge centered at the front of the tree versus the $90^\circ$ wedge centered at the back?
  5. Now it's time to string the lights spiraling around the tree! If I want the lights spaced six inches apart between every full turn, what's a parametric equation for the lights? Plot this on the tree as a parametric curve with ParametricPlot3D and include an image.
  6. What is the length of the spiral of lights?
  7. If I have 90 feet of lights, how far apart should I space each rotation around the tree if I want to space them evenly?
